CONTRAINTE REFERENTIELLE

    UNE CONTRAINTE D’INTEGRITE REFERENTIELLE PORTANT SUR UNE TABLE T1 CONSISTE A IMPOSER QUE LA VALEUR D’UN GROUPE D’ATTRIBUTS APPARAISSE COMME VALEUR DE CLE DANS UNE AUTRE TABLE T2.

    VOYAGE (REFV:ENTIER, DEST:TEXTE, DUREE :ENTIER, TYPE:TEXTE,

    COUT:REEL)

    ADHERENT (NUMA: ENTIER, NOM: TEXTE, PRENOM: TEXTE,

    VILLE: TEXTE)

    RESERVATION (NDOSSIER: ENTIER, NUMA: ENTIER, REFV:ENTIER,

    DATEDEP: DATE, ACOMPTE: REEL)

    DEPART (REFV:ENTIER, DATEDEP: DATE)

    La table RESERVATION a 2 clés étrangères:

    • L’attribut NUMA référence la clé de la table ADHERENT.
    • Le groupe d’attributs REFV, DATEDEP référence la clé de la table DEPART.
